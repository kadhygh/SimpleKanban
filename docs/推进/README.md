# 推进目录说明

这个目录用于存放项目推进过程中的阶段性文档。

## 目录约定

- `v1/`：存放所有与 v1 相关的文档
- 后续如果进入新阶段，可以继续增加：
  - `v2/`
  - `v3/`
  - `research/`
  - `records/`

## 当前规则

- 与 v1 需求、任务拆解、接口设计、实现记录、问题清单、验收标准有关的内容，统一放在 `docs/推进/v1/`。
- `docs/dashboard.md` 用来展示全局推进状态。
- `prepare.md` 用来承载当前阶段的总体准备方案，不负责记录日常推进细节。

## 当前入口

- `docs/推进/v1/README.md`
- `docs/推进/v1/清单.md`
