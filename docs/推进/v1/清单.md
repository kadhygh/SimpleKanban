# v1 清单

## 1. v1 目标

v1 目标是先完成一个本地优先的网页终端闭环，而不是完整 TaskCard 平台。

当前最小闭环：

1. 选择工程目录。
2. 创建或恢复网页终端会话。
3. 在网页中完成终端输入 / 输出交互。
4. 在终端中执行至少一种可配置执行器。
5. 页面刷新后恢复当前会话状态。

## 2. 当前默认技术方向

- 前端：`React + Vite + TypeScript + xterm.js`
- 后端：`Node.js + Fastify/Express + WebSocket + node-pty`
- 模型核心：`Project + TerminalSession + ExecutorProfile`

## 3. 任务拆解

### A. 仓库与工程骨架
- [ ] 确定前后端目录结构
- [ ] 确定包管理方式
- [ ] 确定开发启动命令
- [ ] 确定环境变量和本地配置文件位置

### B. 项目选择链路
- [ ] 设计当前项目数据结构
- [ ] 实现获取当前项目接口
- [ ] 实现选择项目接口
- [ ] 实现项目路径本地持久化
- [ ] 页面展示当前项目路径

### C. 网页终端最小闭环
- [ ] 引入 `xterm.js`
- [ ] 搭建终端展示区域
- [ ] 接入 `node-pty`
- [ ] 在指定项目目录启动终端会话
- [ ] 打通终端输出推送到前端
- [ ] 打通前端输入写回终端
- [ ] 支持终端 `resize`

### D. 会话管理
- [ ] 设计 `TerminalSession` 数据结构
- [ ] 实现创建终端会话
- [ ] 实现获取当前终端快照
- [ ] 实现恢复当前终端会话
- [ ] 防止同项目重复创建活跃终端
- [ ] 记录会话状态和最后输出时间

### E. 执行器模型
- [ ] 设计 `ExecutorProfile` 数据结构
- [ ] 增加默认执行器注册表
- [ ] 支持获取执行器列表
- [ ] 支持预览拼接后的命令
- [ ] 支持将执行器命令注入当前终端

### F. 前端交互层
- [ ] 增加执行器选择器
- [ ] 增加参数输入区
- [ ] 增加状态栏
- [ ] 增加终端连接状态提示
- [ ] 处理页面刷新后的恢复逻辑

### G. 验收
- [ ] 可以选择项目目录
- [ ] 可以创建网页终端
- [ ] 可以正常输入命令并获得输出
- [ ] 可以通过执行器注入命令
- [ ] 页面刷新后可以恢复当前会话状态
- [ ] 会话异常时状态可见

## 4. 先后顺序建议

推荐按以下顺序推进：

1. `A. 仓库与工程骨架`
2. `B. 项目选择链路`
3. `C. 网页终端最小闭环`
4. `D. 会话管理`
5. `E. 执行器模型`
6. `F. 前端交互层`
7. `G. 验收`
