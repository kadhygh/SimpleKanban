# v1 清单

## 1. v1 目标

v1 目标是先完成一个本地优先的网页终端闭环，而不是完整 TaskCard 平台。

当前最小闭环：

1. 选择工程目录。
2. 创建或恢复网页终端会话。
3. 在网页中完成终端输入 / 输出交互。
4. 在终端中执行至少一种可配置执行器。
5. 页面刷新后恢复当前会话状态。

## 2. 当前默认技术方向

- M1 当前实现：零依赖 `Node.js` 本地服务 + 原生 HTML/CSS/JS 页面
- M2 当前落地方向：继续保留现有原生页面与本地 Node 服务，先渐进接入 `xterm.js`
- M2 当前后端方向：在现有本地服务上先接入 `WebSocket + node-pty`
- 后续演进方向：再视需要过渡到 `React + Vite + TypeScript`
- 模型核心：`Project + TerminalSession + ExecutorProfile`

- `M1` 已由你本人手动验证通过，当前可视为本阶段已完成。
- `M2` 最小网页终端闭环已由你本人完成首轮验证：终端可创建、可输入输出、`resize` 复验通过。
- `M2` 当前仍未完全结束，后续若继续补“刷新恢复”等能力，需要继续按人工验证闸门推进。

## 6. 当前实现说明

M1 当前已经完成的是“本地服务 + 当前项目选择 + 页面展示”这一层，并且已经由你本人完成手动确认。

M2 当前正在这套骨架上继续演进，优先目标不是切换技术栈，而是先把网页终端最小闭环打通。

为了先把链路跑通，当前代码采用：

- `source/server/server.mjs`：本地 HTTP 服务与 API
- `source/server/lib/project-store.mjs`：项目状态保存
- `source/server/lib/folder-dialog.mjs`：Windows 文件夹选择对话框
- `source/web/index.html`：M1 页面骨架
- `source/web/app.js`：前端交互逻辑
- `source/web/styles.css`：页面样式

当前进入 M2 时，优先采用“在现有代码上渐进接入”的策略：

- 前端继续保留原生 HTML/CSS/JS 页面；
- 后端继续保留当前本地 Node 服务；
- 在此基础上补齐终端会话、双向通信和状态展示；
- 待最小闭环稳定后，再决定是否切换到更完整的前端技术栈。
