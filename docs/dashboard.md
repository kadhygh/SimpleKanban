# SimpleKanban Dashboard

最后更新：2026-03-07

## 1. 项目定位

SimpleKanban 是一个面向 Unity 本地开发流程的任务与执行管理器。

当前阶段不追求完整协作框架，而是优先打通：

- 本地项目绑定
- 网页终端会话
- CLI 交互与状态可见
- 可扩展执行器模型

## 3. 当前锁定决策

- v1 以 **网页托管终端** 为核心，而不是外部 terminal 窗口。
- v1 以 **单机 / 单项目 / 单活跃终端** 为最小目标。
- 后续一个 `TaskCard` 可以关联多个可执行入口，因此底层先抽象 `ExecutorProfile`。
- `docs/rescource` 中的 `vibekanban` review 可作为参考，但不是必须照搬的实现依据。
- 当前 `M2` 继续采用渐进式实现：先在现有骨架上接入终端能力，再决定是否升级前端技术栈。

## 3.1 验证策略

从 2026-03-07 起，所有里程碑都按“**先由你本人验证，再继续推进**”执行。

状态口径统一为：

- 已完成：你本人已实际验证通过；
- 进行中：功能开发中，或代码已实现但仍待你验证；
- 未开始：尚未进入实现。

补充约定：

- 助手每完成一个最小步骤，都需要先停下并给出你的验证操作与预期结果；
- 只有你明确回复“验证通过”后，才继续推进下一步；
- 开发侧自测不能替代你的最终验收。
